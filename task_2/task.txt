Задача читателей/писателей:
- Имеется N читателей и M писателей:
  - читатель - поток исполнения, стремящийся считать содержимое разделяемого файла и вывести на консоль в виде f'Reader {reader_id}: {file_contents}';
  - писатель - поток исполнения, стремящийся записать в разделяемый файл строку (свой ID и состояние внутреннего счетчика)(файл перезаписывается), нарастить значение счетчика и уснуть на случайный отрезок времени < T.
- Доступ к разделяемому файлу осуществляется по следующим правилам:
  - в единый момент времени может происходить либо чтение, либо запись;
  - одновременно файл могут читать n читателей;
  - одновременно в файл может писать только один писатель;
  - если какой-либо писатель ожидает своей очереди, читатели не должны читать (у писателя приоритет).
- Выполнить задачу используя потоки и процессы.
- Оформить в виде утилиты командной строки:
    - Утилита должна неаварийно завершать работу по нажатию Ctrl+C.
    - Должен иметься способ задания параметров N, M, n и T из командной строки (argparse).
    - В качестве тестовых значений можно использовать N = 10, M = 2, n = 3, T = 50ms.
